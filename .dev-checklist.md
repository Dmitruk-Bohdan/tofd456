# Чеклист для проверки изменений

## Перед изменением файлов:
- [ ] Проверить оригинальную структуру файла
- [ ] Проверить версии зависимостей в других частях проекта
- [ ] Понять, как работает текущая реализация

## После изменения:
- [ ] Проверить синтаксис (linter)
- [ ] Проверить типы (TypeScript)
- [ ] Убедиться, что изменения совместимы с существующим кодом
- [ ] Проверить, что не сломаны существующие функции

## Для IDL/контрактов:
- [ ] НЕ изменять структуру IDL вручную - использовать оригинальный из target/idl
- [ ] Проверить версию Anchor в client и в корне проекта
- [ ] Убедиться, что IDL синхронизирован с контрактом

## Для зависимостей:
- [ ] Проверить версии в package.json корня проекта
- [ ] Синхронизировать версии между client и server
- [ ] Проверить совместимость версий
